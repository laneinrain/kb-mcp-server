# Requirements: kb-mcp-server

**Defined:** 2026-07-20  
**Milestone:** v1.6 Web Usability — Model Settings & MCP User Guide  
**Core Value:** An MCP client can reliably semantic-search ingested documents through a stable tool interface.

## Milestone Goal

提升 Web 管理端易用性：**(1)** 在「设置」页可配置 embedding / rerank 模型（含开关与召回条数），不再仅依赖 `.env`；**(2)** 「使用说明」升级为可操作的 User Guide，覆盖 **Cursor / CodeBuddy** 在 JWT 用户认证开启时通过 MCP 访问知识库的完整步骤。

## v1.6 Requirements

### Model Settings (runtime)

- [x] **CONF-04**: `SettingsStore` 持久化检索模型配置：`embeddingModel`、`rerankEnabled`、`rerankModel`、`rerankCandidates`；首次启动从 env（`EMBEDDING_MODEL` / `RERANK_*`）种子写入
- [x] **CONF-05**: REST `GET /api/v1/settings` 返回 `models` 分组；新增 `PATCH /api/v1/settings/models` 校验并更新上述字段（`rerankCandidates` 仍为 1–50）
- [x] **CONF-06**: `EmbeddingClient` / `RerankClient` / `SearchService`（及入库路径）运行时读取 settings 中的模型配置，而非仅启动时冻结的 env 常量
- [x] **CONF-07**: Web「设置」页增加「Embedding / Rerank」表单：可编辑模型 ID、rerank 开关、候选条数；保存后立即对后续搜索生效
- [x] **CONF-08**: 变更 embedding 模型时 UI 与 API 文档明确风险提示（已有向量与模型不一致时需重新入库）；`embeddingDimensions` **只读展示**（仍由 env `EMBEDDING_DIMENSIONS` 决定，本里程碑不支持在线改维度）

### MCP User Guide (Web Help)

- [x] **DOCS-01**: 「使用说明」从简短步骤列表升级为结构化 User Guide（中文）：上传/搜索、鉴权概念、MCP 工具一览
- [x] **DOCS-02**: 文档化 **Cursor** 在 `USER_AUTH_ENABLED=true` 时的 MCP 接入：获取 JWT → HTTP `mcp.json` `headers.Authorization: Bearer <jwt>` → 或 stdio `MCP_USER_TOKEN`
- [x] **DOCS-03**: 文档化 **CodeBuddy**（或同类 MCP 客户端）同等 JWT 流程：HTTP URL + Bearer，以及本地 stdio 等价配置说明
- [x] **DOCS-04**: Help（或设置旁）提供「复制当前登录 JWT」操作，便于粘贴到客户端 MCP 配置（仅当前会话 token；标注过期与安全注意）
- [x] **DOCS-05**: User Guide 写明鉴权矩阵摘要：JWT → 用户文档集；`API_KEY` → 全局服务账号；无凭据 → 401 / 启动失败（与 README v1.5 一致）

## Out of Scope (v1.6)

- Hybrid BM25 + semantic fusion (RETR-01)
- 在线修改 `EMBEDDING_DIMENSIONS` 或自动重建 Chroma collection
- 完整「MCP Token 管理」产品（长期 token / 吊销列表）— 仅复制当前 JWT
- 生产 CasAuthProvider 实现
- 多用户 stdio 同进程会话
- Rerank evaluation harness
- User account delete/disable

## Key Decisions (locked for planning)

| Decision | Rationale |
|----------|-----------|
| 模型配置进 SQLite settings（与 context 同模式） | 关闭「Web rerank 仅 env」缺口；运维可热改 |
| `embeddingDimensions` 只读 | 改维度需重建向量；避免静默损坏索引 |
| Embedding 模型可改但强制风险文案 | 允许换 CherryIn 模型名；用户自行决定是否重入库 |
| 不新建独立「模型」页 — 挂在现有「设置」 | 减少导航噪音，符合「增强易用性」 |
| User Guide 放在 Web Help，可复制 JWT | 解决 v1.5 defer「Web MCP token UI」的最小可用切片 |
| Cursor + CodeBuddy 双客户端 | 用户明确要求；配置形态均为 HTTP Bearer 或 stdio env |
| 运行时读 settings，非仅 boot env | 保存后无需重启 Backend/MCP 即可影响新请求（同进程内） |
| Embedding + Rerank 均为 admin-only（D-13/D-14） | 模型变更影响全局检索质量；非 admin 只读展示 |

## Target Flows

### Model settings

```
GET /api/v1/settings → { chunk, context, models }
PATCH /api/v1/settings/models → persist SQLite (admin when USER_AUTH_ENABLED)
SearchService.search / ingest embed → settingsStore.getModelConfig()
```

### MCP JWT (Cursor / CodeBuddy)

```
Web 登录 → 复制 accessToken
  → Cursor/CodeBuddy mcp.json:
      url: http://127.0.0.1:3100/mcp
      headers: { Authorization: "Bearer <jwt>" }
  → 或 stdio: MCP_USER_TOKEN=<jwt>
工具 search_knowledge / read_around / read_file → 用户文档 ACL
```
