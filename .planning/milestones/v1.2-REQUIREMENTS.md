# Requirements: kb-mcp-server

**Defined:** 2026-07-05  
**Milestone:** v1.2 Multi-User Auth & Hash Upload  
**Core Value:** An MCP client can reliably semantic-search ingested documents through a stable tool interface.

## v1.2 Requirements

### Auth Center

- [x] **AUTH-01**: `@kb/auth` package exposes `AuthProvider` with MockCasAuthProvider; production swap via config
- [x] **AUTH-02**: User store in SQLite; mock CAS validates credentials (no plaintext passwords) — adapted from local bcrypt register spec
- [x] **AUTH-03**: Login REST endpoint issues signed JWT with configurable expiry
- [x] **AUTH-04**: Backend accepts user JWT; legacy `API_KEY` bearer retained for service/CLI
- [x] **AUTH-05**: Invalid/missing token returns 401 with consistent error body
- [x] **AUTH-06**: Auth module documents swap points in package README

### Multi-User Documents

- [x] **USER-01**: `documents.user_id`; Chroma chunk metadata includes `userId`
- [x] **USER-02**: Users list/upload/delete/search own documents (+ shared system legacy)
- [x] **USER-03**: Legacy documents migrate to system user on startup
- [x] **USER-04**: Unauthenticated protected routes return 401 when auth enabled

### Web Auth UI

- [x] **WEB-01**: Login page (简体中文) stores JWT for API calls
- [ ] **WEB-02**: Register page — **DEFERRED** (JIT login only; see Phase 8 D-05)
- [x] **WEB-03**: Protected admin routes redirect to login
- [x] **WEB-04**: Logout clears token and returns to login

### Hash Dedup Upload

- [x] **INGE-10**: `content_hash` (SHA-256 normalized parsed text) stored on documents
- [x] **INGE-11**: Same `(user_id, filename)` + identical hash → unchanged, no re-embed
- [x] **INGE-12**: Same pair + different hash → replace vectors and re-ingest
- [x] **INGE-13**: Upload response includes `outcome`: created | unchanged | replaced
