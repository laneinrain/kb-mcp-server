# Milestone v1.4: Qwen Rerank Search

**Status:** ✅ SHIPPED 2026-07-16  
**Phases:** 13–15  
**Total Plans:** 7

## Overview

Improve search precision with two-stage retrieval: Chroma vector recall (`RERANK_CANDIDATES` default 30) followed by CherryIn Qwen3 rerank (`qwen/qwen3-reranker-0.6b`). Applies to REST search and MCP `search_knowledge`. Graceful fallback to vector-only when disabled or API unavailable.

## Milestone Summary

**Resolves from prior milestones:**
- RETR-02 cross-encoder reranking (deferred since v1.0)
- Retrieval precision gap on large documents / noisy top-k

**Key Decisions:**
- Model `qwen/qwen3-reranker-0.6b` via CherryIn (same API key as embeddings)
- Default `RERANK_ENABLED=true`, `RERANK_CANDIDATES=30`
- Vector fallback on rerank API failure

**Issues Deferred:**
- RETR-01 hybrid BM25
- Runtime rerank toggle in Web settings
- Rerank evaluation harness (manual UAT only)
