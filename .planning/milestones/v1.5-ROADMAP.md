# Milestone v1.5: MCP User Isolation

**Status:** ✅ READY TO SHIP (phases complete)  
**Phases:** 16–18  
**Total Plans:** 9

## Overview

Scope MCP tools to the authenticated user's document set. HTTP transport validates Bearer JWT (or service `API_KEY`) on `/mcp` requests; stdio binds a single user via `MCP_USER_TOKEN`. All three tools (`search_knowledge`, `read_around`, `read_file`) enforce the same ACL as REST search. Legacy global mode preserved when `USER_AUTH_ENABLED=false`.

## Milestone Summary

**Resolves from prior milestones:**
- PLAT-04 per-user MCP auth (deferred since v1.0)
- MCP/REST isolation parity gap

**Key Decisions:**
- Reuse REST JWT + `API_KEY` composite auth
- HTTP Bearer per request; stdio `MCP_USER_TOKEN` per process
- `ContextService` ACL mirrors `SearchService`
- Fail closed when auth required

**Issues Deferred:**

- Multi-user stdio sessions
