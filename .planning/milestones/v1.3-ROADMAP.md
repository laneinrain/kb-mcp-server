# Milestone v1.3: Mock CAS Admin Console

**Status:** 🚧 PLANNED  
**Phases:** 10–12  
**Total Plans:** 9 (estimated)

## Overview

Extend mock-mode authentication with local registration, a built-in administrator (工号 `00000` / `admin123`), admin-only REST APIs for account and cross-user document management, and a 简体中文 Web admin console. Production CAS path (`CAS_MOCK=false`) remains unchanged — admin features are disabled.

## Phases

### Phase 10: Mock Local Auth & Admin Bootstrap

**Goal:** `@kb/auth` supports `local` users with bcrypt, admin bootstrap, register API, and JWT roles — gated to `CAS_MOCK=true`.

**Requirements:** ADMIN-01, ADMIN-02, AUTH-07, AUTH-08, AUTH-09, AUTH-10

**Plans:** 3/3 planned

- [ ] 10-01: Schema `role` + bcrypt + UserStore `ensureAdminUser` / `registerLocalUser`
- [ ] 10-02: MockCasAuthProvider local login + register + JWT role claim
- [ ] 10-03: Admin bootstrap in services + `POST /api/v1/auth/register` + docs

**Cross-cutting constraints:**
- Admin 工号 `00000` reserved; system user `00000000` remains non-loginable
- `CasAuthProvider` stub untouched

---

### Phase 11: Admin REST API

**Goal:** Admin-only endpoints for user directory and cross-user document CRUD.

**Requirements:** USER-05, USER-06, USER-07, USER-08, USER-09

**Plans:** 3/3 complete

- [x] 11-01: `createAdminRouteOpts()` + `GET /api/v1/admin/users`
- [x] 11-02: Admin document list/get/delete routes
- [x] 11-03: Upload on behalf + tests

**Cross-cutting constraints:**
- Regular JWT user routes unchanged (USER-09)
- API_KEY service mode retains global access (no role check)
- Admin cannot delete system user account (docs only)

---

### Phase 12: Web Admin Console

**Goal:** Register page and admin 用户管理 UI in 简体中文.

**Requirements:** WEB-02, WEB-05, WEB-06, WEB-07, WEB-08

**Plans:** 3/3 complete

- [x] 12-01: RegisterPage + routing + register availability probe
- [x] 12-02: Admin 用户管理 tab + user list
- [x] 12-03: Per-user document panel (upload/list/delete)

**Cross-cutting constraints:**
- Reuse existing UploadPanel/DocumentTable patterns where possible
- Admin tab only when JWT user.role === `admin`
- Login page links to register when mock mode
