<interfaces>
From RESEARCH — AuthProvider contract:
```typescript
export interface AuthProvider {
  register(input: RegisterInput): Promise<AuthUser>;
  login(input: LoginInput): Promise<{ user: AuthUser; tokens: AuthTokens }>;
  validateAccessToken(token: string): Promise<AuthUser>;
  getUserById(id: string): Promise<AuthUser | null>;
}
```

users table columns: id (uuid), username (unique nocase), email (nullable unique), password_hash, created_at
</interfaces>
</context>

<threat_model>
| Threat | Severity | Mitigation in this plan |
|--------|----------|-------------------------|
| Plaintext password storage | High | bcryptjs hash only in SQLite password_hash column |
| Weak JWT secret | High | JWT_SECRET min 32 chars enforced in Zod when USER_AUTH_ENABLED |
| Token forgery | High | jose HS256 verify with JWT_SECRET |
| SQL injection | Medium | better-sqlite3 prepared statements only |
| User enumeration on register | Low | Phase 7 returns 409 on duplicate username (acceptable for admin KB) |
</threat_model>
