# Phase 4 — UI Design Contract

> Visual and interaction contract for the kb-mcp-server admin SPA (`apps/web`). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope:** Web admin only (WEB-01–WEB-04). CLI is out of UI scope.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — native HTML + minimal CSS modules or a single `App.css` |
| Icon library | none — text labels and Unicode symbols only (e.g. `×`, `⚠`) |
| Font | System stack: `ui-sans-serif, system-ui, -apple-system, Segoe UI, Roboto, sans-serif`; monospace for IDs/scores: `ui-monospace, SFMono-Regular, Consolas, monospace` |

**Rationale:** User and CONTEXT lock a local-first dev-tool aesthetic without shadcn/Tailwind. No `components.json` in repo; shadcn initialization deferred by explicit choice. *(Source: user prompt, 04-CONTEXT.md Web Stack)*

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon/text gaps, table cell tight padding |
| sm | 8px | Button padding vertical, compact row gaps |
| md | 16px | Default padding inside panels, form field gaps |
| lg | 24px | Tab bar padding, section vertical rhythm |
| xl | 32px | Main content max-width gutters, upload zone min-height padding |
| 2xl | 48px | Page header to content gap |
| 3xl | 64px | Not used in Phase 4 — reserve for future settings page |

Exceptions:

- Upload drop zone minimum height: **120px** (touch-friendly target without full 44px button rule elsewhere)
- Delete confirm modal actions: **44px** min tap height on primary/cancel buttons
- Table action column width: **80px** fixed

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 600 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 24px | 600 | 1.2 |

**Usage:**

- **Display:** Page title `kb admin` in header (once)
- **Heading:** Panel titles (`Documents`, `Search`), modal title (`API key required`)
- **Body:** Table cells, result snippets, form inputs, button text
- **Label:** Column headers, field labels (`Query`, `Top K`), status badges, score metadata

Monospace at **12px / 400 / 1.4** for `documentId` truncation display and numeric `score` (not a fifth size — variant of Label).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (upload) | **Upload file** |
| Primary CTA (search) | **Run search** |
| Empty state heading (documents) | **No documents yet** |
| Empty state body (documents) | **Upload a .txt, .md, or text-layer PDF to index it in the knowledge base.** |
| Empty state heading (search results) | **No results** |
| Empty state body (search results) | **Enter a query and run search to preview ranked chunks from the corpus.** |
| Error state (generic API) | **Request failed:** `{message from API or "Unknown error"}` — **Check the backend is running and try again.** |
| Error state (upload 415) | **Unsupported file type.** — **Use .txt, .md, .markdown, or .pdf with extractable text.** |
| Error state (network) | **Cannot reach the API.** — **Start the backend (`pnpm dev`) and refresh.** |
| Destructive confirmation | **Delete document:** **Remove "{filename}" and all its vectors? This cannot be undone.** |
| API key modal title | **API key required** |
| API key modal body | **Enter the API key configured on the server (AUTH_ENABLED). Stored for this browser tab session only.** |
| API key modal CTA | **Save key** |
| API key modal cancel | **Cancel** (closes modal; failed request remains in error state) |
| Upload in progress | **Indexing…** (disable upload control; subtitle: **Large PDFs may take a minute.**) |

## Layout & Navigation

**Pattern:** Single-page app with **horizontal tab bar** — not separate routes required for Phase 4.

```
┌─────────────────────────────────────────────────────────────┐
│  kb admin                                    [optional badge]│
├─────────────────────────────────────────────────────────────┤
│  [ Documents ]    [ Search ]     ← active tab: accent underline
├─────────────────────────────────────────────────────────────┤
│  {active panel content — max-width 960px, centered}        │
└─────────────────────────────────────────────────────────────┘
```

| Area | Specification |
|------|---------------|
| Viewport | Min width **320px**; content max-width **960px**, horizontal auto margin |
| Header | Height **48px**; Display title left; no nav links beyond tabs |
| Tab bar | Secondary surface (`#ffffff`); bottom border `#e4e4e7`; active tab **2px** accent bottom border |
| Default tab | **Documents** on load |
| Tab state | Persist active tab in React state only (no URL routing required) |

**Documents panel** (top to bottom, **24px** gap between blocks):

1. Upload zone (WEB-01)
2. Documents table (WEB-02 + WEB-03)

**Search panel:**

1. Search form (query + topK)
2. Results list (WEB-04)

## Component Inventory

### 1. `AppShell`

- Renders header, tab bar, active panel
- Props: `activeTab`, `onTabChange`

### 2. `UploadPanel` (WEB-01)

| Property | Value |
|----------|-------|
| Input | Hidden `<input type="file" accept=".txt,.md,.markdown,.pdf">` |
| Trigger | Click anywhere on drop zone **or** button **Choose file** |
| Drag-drop | `dragover` → accent dashed border; `drop` → single file (first file if multiple) |
| Allowed | `.txt`, `.md`, `.markdown`, `.pdf` — client-side filter before POST; show 415 copy if server rejects |
| Submit | Auto-upload on file select/drop (no separate file list queue in v1) |
| Loading | Replace zone content with spinner text **Indexing…**; disable further drops |
| Success | Toast-free: invalidate document list query; optional inline success **Uploaded {filename}** for **3s** |
| API | `POST /api/v1/documents` multipart field `file` |

### 3. `DocumentTable` (WEB-02, WEB-03)

| Column | Source field | Display |
|--------|--------------|---------|
| Filename | `filename` | Body, truncate with ellipsis at **40ch**, full text in `title` tooltip |
| Status | `status` | Badge: `indexed` green, `pending`/`processing` amber, `failed` red |
| Chunks | `chunkCount` | Body, right-aligned number |
| Collection | `collection` | Label size, muted if default |
| Updated | `updatedAt` | Label size muted, locale `YYYY-MM-DD HH:mm` |
| Actions | — | Text button **Delete** (destructive color) |

- Sort: server order (registry list order); no client sort in Phase 4
- Row hover: `#fafafa` background
- Empty: copywriting empty state centered above table area

**Delete flow (WEB-03):**

1. Click **Delete** → native `<dialog>` or fixed overlay modal (not `window.confirm`)
2. Show destructive confirmation copy with `{filename}`
3. Buttons: **Cancel** (secondary outline) | **Delete** (destructive fill)
4. On confirm: `DELETE /api/v1/documents/:id`; pending disables both buttons
5. Success: close modal, invalidate list; no toast
6. Error: show error copy inside modal, keep open

### 4. `SearchPanel` (WEB-04)

**Form:**

| Field | Type | Default | Constraints |
|-------|------|---------|-------------|
| Query | `<textarea rows={3}>` or `<input>` | empty | Required, max **2000** chars (match API) |
| Top K | `<select>` | **5** | Options **1–10** integer |

- Submit: **Run search** button; Enter in query field submits if focus in form
- Disable submit when query empty or mutation pending

**Results list:**

Each result card (secondary surface, **16px** padding, **8px** gap between cards):

| Element | Display |
|---------|---------|
| Score | Monospace **12px**, muted — format `{score.toFixed(3)}` |
| Filename + chunk | Label — `{filename} · chunk {chunkIndex}` |
| Snippet | Body — full `text` with `white-space: pre-wrap`; max-height **120px** overflow auto |
| Document ID | Monospace muted **12px** — truncate middle with ellipsis |

- Order: API order (ranked by score descending)
- Empty results array: empty state copy for search
- Loading: **Searching…** below form

**API:** `POST /api/v1/search` body `{ query, topK }`

### 5. `ApiKeyModal` (CONF-03 web UX)

| Property | Value |
|----------|-------|
| Trigger | First `401` from any API call via shared fetch client |
| Storage key | `sessionStorage` key `kb_api_key` |
| Field | `<input type="password" autocomplete="off">` placeholder **API key** |
| Save | Writes to sessionStorage, closes modal, **retries the failed request once** |
| Cancel | Closes modal; user can retry manually |
| Re-prompt | On subsequent 401 (wrong key): reopen modal with inline error **Invalid API key** |
| Header injection | `Authorization: Bearer ${key}` on all `/api/v1/*` requests when key present |

Modal: centered, **400px** max-width, **24px** padding, backdrop `#00000066`.

### 6. Shared `ApiClient` + React Query

- Base URL: relative `/api` (Vite proxy dev, same-origin prod)
- Queries: `['documents']` list; mutations: upload, delete, search
- Invalidate `['documents']` after successful upload and delete
- Global error boundary: not required; inline error regions per panel

## Accessibility (minimal)

- Tab bar: `role="tablist"`, tabs `role="tab"`, panels `role="tabpanel"`, `aria-selected` on active tab
- Delete modal: focus trap, `aria-modal="true"`, initial focus on Cancel
- Upload zone: `aria-label="Upload document"`; announce **Indexing…** via `aria-live="polite"` region
- Form labels: explicit `<label htmlFor>` for query and topK
- Color contrast: text on backgrounds meets WCAG AA for body and labels
